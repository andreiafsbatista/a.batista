Background:

The "Display controller" is a specific part of the Acro Companion application, within the scoring solution offered by AC. It allows real-time control of the information being sent to the different screens during a competition.

Problems:

The existing screen, created as a MVP with the minimum functionality in mind, lacked a human-centred perspective; ter a certeza que a informação displayed é a correta e facilmente corrigir quando necessário durante uma fast-paced competition.

Goal:

test test test

What is "Display controller" and who uses it

The display controller is a feature that let's competition organizers create and configure of the different screens that will display information during the competition. Creation of automatic displays that switch between a group of screens. During the competition, it's used to kee track of the automatic screens, override the loop to send in to correct information or to send in screens that can't be automated (pause, podium...)

Before redesign - Client feedback

Business perspective - the first iteration served the purpose of having the functionality and test with users to adress the usefulness and receive a first roud of feedback of new things to add. and the idea was to invest in a user-friendly interface.

Main problems found

Based on the feedback and the current State Evaluation (problems the screens has)

  • Users complained about the complexity of the screen
  • A lot of buttons (too much information)
  • A lot of clicks to trigger a screen
  • Can't see the status of the changes
  • More personalization
  • The competition stage can be huge, it's difficult to keep track of what's being displayed on all screens
  • Different tabs to set the screens (back and forward)??????
